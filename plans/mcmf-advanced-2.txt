1. Напоминание.
    - Остаточная сеть
    - Поток оптимальный = нет циклов
    - Базовый алгоритм. Ищем кратчайший путь => циклов не появляется [O(SP * |F|)] 
    - Потенциалы w'_ij = w_ij + p(j) - p(i)
    - Использование Дейкстры вместо Форда-Белмана 
2. Capacity scaling [ 30-40 минут ]
    - LR-like сведение циркуляции к минкост потоку с положительными ребрами
    - Навешиваем масштабирование [O(E * logMax * SP)]
	      + Сохраняем потенциалы между итерациями
	      + После ослабления условия могли появиться отрицательные ребра
	      + Надо это починить с помощью 2a
3. Cost scaling [V^2Elog(V*MAXC)]
    - Определения, избыток, недостаток.
    - Лемма, что из избытка есть путь в остаточной сети в недосаток.
    - Сведение потока к циркуляции, eps-оптимальная циркуляция
    - Лемма, что 1/(n+1)-оптимальная циркуляция - оптимальная
    - Алгоритм для eps -> eps/2
        + Если есть отрицательное ребро можно по нему протолкнуть 
        + Если нет отрицательных ребер, но есть избыток можно увеличить потенциал
        + Если ничего сделать нельзя -> мы eps/2 оптимальный поток
        + while (true) делаем что-нибудь
        + Каждая вершина будет поднята не более 3n раз, т.к. иначе путь из 3b слишом отрицательный
        + Значит не более VE насыщающих проталкиваний
        + Значит не более V^2E ненасыщающих проталкиваний
        + Можно сделать lift-to-front будет V^3 ненасыщающих проталкиваний
    - Отвлечение про preflow-push
4. Поиск цикла минимального среднего веса
    - Посчитали d[v][k] - кратчайший путь до v длины k 
    - Ответ это: min v max k (d[v][n] - d[v][k]) / (n - k)
    - Все линейно, по этому можно докзаывать для цикла длины 0. 
    - Меньше не может быть, потому что нет отрицательных циклов, а значит путь длины n не может быть меньше чем все
    - Больше не может быть, потому что до всех вершин кратчайший путь можно выбрать по циклу, и есть такая для которой будет нужный сдвиг
    - Сам цикл можно найти пойдя по предками из вершины для которой достигся минимум
 5. Отмена циклов минимального среднего веса 
    - Только общий план доказательства, скорее надо вообще всю тему выпилить
    - Минимальный средний цикл = min max потенциала = eps из eps-оптимальности
    - O(VElogMAX * Find-cycle) 
        + каждые E итераций вес минимального eps уменьшается на (1 - 1/V)
    - O(VE^2logV * Find-cycle)
        + После блока из VElogV операций одно ребро никогда не поменяется
